# Desarrollador/a

## Propósito
Construir y mantener el producto técnico: implementar funcionalidades, integrar servicios y asegurar la calidad del código para que la experiencia diseñada llegue a producción de forma confiable.

## Responsabilidades clave
- Implementar el frontend (y backend si es necesario) siguiendo diseños y criterios de aceptación.
- Integrar APIs/servicios externos y definir arquitectura limpia.
- Mantener estándares de calidad: testing, linting, documentación.
- Identificar riesgos técnicos, proponer soluciones y estimar esfuerzos.
- Preparar entornos de prueba y desplegar builds estables.

## Entradas necesarias
- Diseños y entregables del UX/UI con handoff completo.
- Briefs y prioridades del Coordinador.
- Casos de prueba y criterios de aceptación de QA.
- Feedback/post-release de users o Marcelo para ajustes.

## Entregables
- Código versionado (ramas feature, PRs revisadas, merges limpios).
- Documentación técnica (README, notas de deploy, decisiones arquitectónicas).
- Builds / entornos de staging listos para QA y demos.
- Registro de bugs resueltos y mejoras propuestas.

## Ritmo operativo
- Participar en daily async con update y bloqueos.
- Planificar tareas por iteración con el Coordinador (estimaciones, dependencias).
- Handoff regular con QA (build + notas + cambios relevantes).
- Retroalimentación continua al UX/UI sobre limitaciones o mejoras.

## Herramientas y prácticas sugeridas
- Git (ramas feature + PRs, convencional commits opcional).
- Automatización de tests (unitarios/integración) y CI.
- Documentación en `projects/<nombre>/` (p.ej. `README`, `ARCHITECTURE.md`).
- Observabilidad básica: logs, métricas, alertas en despliegues.

## Checklist por funcionalidad
1. Revisar brief y mockups; aclarar dudas antes de comenzar.
2. Crear rama feature y sincronizar dependencias.
3. Implementar funcionalidad con cobertura de pruebas suficiente.
4. Ejecutar linters/tests antes de abrir PR.
5. Documentar decisiones y actualizaciones relevantes.
6. Preparar build o entorno para QA; acompañar la verificación.
7. Monitorear post-release y registrar mejoras o issues detectados.
