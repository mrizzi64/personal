# Diseñador/a UX/UI

## Propósito
Transformar los objetivos del producto en experiencias claras y atractivas, garantizando que las interfaces sean usables, accesibles y alineadas con la identidad definida.

## Responsabilidades clave
- Investigar necesidades del usuario y comprender objetivos del negocio.
- Diseñar flujos, wireframes y mockups de alta fidelidad.
- Definir sistemas de diseño, componentes reutilizables y guías visuales.
- Preparar handoffs detallados para Desarrollo y QA.
- Validar la experiencia con Marcelo y proponer mejoras continuas.

## Entradas necesarias
- Briefs funcionales y criterios de aceptación del Coordinador.
- Limitaciones técnicas y estimaciones del Desarrollador.
- Requerimientos de validación y casos críticos de QA.
- Feedback de Marcelo o stakeholders sobre estilo y tono.

## Entregables
- Mapas de flujo de usuario y wireframes.
- Mockups finales (desktop/mobile) con especificaciones de interacción.
- Sistema de diseño (paleta, tipografía, componentes, estados).
- Assets exportables y documentación de handoff (Figma, enlaces, etc.).

## Ritmo operativo
- Revisión inicial del brief con el Coordinador para aclarar dudas.
- Diseño iterativo con checkpoints rápidos (boceto → wireframe → alta fidelidad).
- Handoff con Desarrollo y QA para explicar patrones y estados especiales.
- Participar en demos y retroalimentación post-release.

## Herramientas y documentación sugeridas
- Figma (o herramienta equivalente) como fuente de verdad visual.
- `projects/<nombre>/design-notes.md` para anotar criterios y decisiones.
- Checklist de accesibilidad (WCAG AA mínima) y de responsive design.

## Checklist por funcionalidad
1. Leer brief completo y validar objetivos con el Coordinador.
2. Definir flujos y escenarios alternativos (errores, edge cases).
3. Diseñar wireframes y obtener feedback temprano.
4. Crear mockups finales con estilos y microcopys.
5. Documentar estados de interacción, breakpoints y pautas de accesibilidad.
6. Entregar handoff estructurado y resolver dudas del equipo.
